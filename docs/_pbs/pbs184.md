## SASS — Syntactically Awesome Style Sheets

The reason SASS exists is to automate the creation of regular cascading style sheets (CSS) by adding powerful features missing from CSS itself. SASS is a so-called *CSS pre-processor*, and is the defector winner in the category, having out-lasted the other early contenders in the space,  most notably LESS (*Leaner Style Sheets*).

The process is simply:

1. Write your style sheets in SASS
2. Compile the SASS stylesheet to CSS
3. Import the compiled CSS into your HTML files using a `<link rel="stylesheet">` tag in `<head>`, or an  `@import`  at-rule in a `<style>` tag.

For historical reasons the SASS pre-processor accepts two different syntaxes, the original indent-based SASS syntax, and the newer curly-braces-based SCSS syntax. The newer SCSS syntax is a super-set of regular CSS, so developers generally find it easier to learn, and because it uses curly braces to group statements rather than relying purely on indentation, it is more robust when copy-and-pasted. **We will only be using the newer SCSS syntax**!

Yes, this is very confusing, we are going to be using a piece of software named SASS that supports two syntaxes, one of which is named the *SASS syntax* and saved in `.sass` files, and to the other is named the *SCSS syntax* and saved in `.scss` files, but both of which are referred to as *SASS files*, and both can be converted to regular CSS with the `sass` command!

To keep things as un-unclear as possible, we're going to utterly ignore the older SASS syntax and only use the newer SCSS syntax, and we're going to **refer to our SASS code in that SCSS syntax as simply _'SASS'_**, like just about the entire internet does these days!

## SASS Basics

Firstly, because we are using the newer syntax, all our code will be saved in files with the `.scss` file extension.

Secondly, because SASS is a super-set of CSS, **all valid CSS code is valid SASS code**!

In addition to being able to do everything CSS can do SASS adds many features that CSS does not support. We're not going to explore anywhere near all of them. Instead, we're going to focus on the ones we're mostly likely to use when working with static site generators like Jekyll, and especially when working with Bootstrap inside Jekyll.

Here's a quick overview of the features we'll be exploring:

* **Selector nesting** — using curly braces to control the targeting of style declarations more clearly
* **SASS Variables** — both much simpler to use and more powerful than CSS custom properties
* **Conditionals and Loops** — implemented as SASS-only at-rules, things like `@if`, `@each` etc..
* **SASS Operators** — like CSS functions, but SASS-only
* **Inheritance** — import the styles from one selector into another to avoid code duplication
* **Mixins** — SASS's implantation of style templates
